## Typography

Sourced from existing component patterns (`HeaderBar.tsx`, `SideNav.tsx`, `globals.css`).

Two weights only: 400 (normal) and 600 (semibold).

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 14px (text-sm) | 400 (normal) | 1.5 | Default text in plugin panes, nav items, status text |
| Label | 12px (text-xs) | 400 (normal) | 1.4 | SSE status, settings labels, metadata, error detail pre block |
| Heading | 16px (text-base) | 600 (semibold) | 1.2 | Plugin pane title in error card, section headings |
| Display | 13px (text-[13px]) | 600 (semibold, uppercase) | 1.0 | Nav group labels (10px, uppercase, wide tracking — `text-[10px] uppercase tracking-wider`). Visual distinction comes from `uppercase` + `tracking-wider`, not from additional weight. |

Note: The existing codebase uses `text-sm` (14px) as the primary body size and `text-xs` (12px) for secondary labels. These are pre-established patterns — do not deviate.

Error card specifically uses:
- Plugin name: 14px, weight 600 (semibold), color `text-red-400`
- "Something went wrong": 14px, weight 400, color `text-neutral-400`
- Retry / Details buttons: 12px, weight 400

Stub plugin header (Phase 1 placeholder only):
- Plugin name: 14px, weight 600 (semibold), color `text-zinc-400`

## Interaction Contract

### Drag and Resize (SHELL-01)
- Drag handle: entire plugin pane header bar (top 36px of pane)
- Resize handle: bottom-right corner grip (react-grid-layout default resize handle)
- During drag: placeholder shown as `border-2 border-accent/40 bg-accent/10 rounded-lg` — no grid lines
- Drag cursor: `cursor-grab` on handle, `cursor-grabbing` during drag
- Adjacent plugins auto-push (no overlap) — no visual snap indicators

### Auto-Save (SHELL-06, D-14)
- Debounce: 2500ms after last drag/resize event ends
- No manual save button in Phase 1 shell
- No save indicator in Phase 1 (silent auto-save)
- On page load: restore from Postgres if workspace exists, else load Daily Ops preset (D-10)

### Error Boundary (SHELL-04, D-16, D-17)
- Error card occupies the full plugin pane at its current size
- Error card layout (top-to-bottom): plugin name (red-400) → "Something went wrong" (neutral-400) → Retry button → Details toggle → error pre block (collapsible)
- Retry resets `hasError` state — full remount of the plugin component
- Details section: collapsible `<pre>` block, max-height 128px (max-h-32), overflow-y auto
- Adjacent plugins: unaffected, continue rendering normally

### Loading State (SHELL-07)
- Grid shell SSR loading placeholder: `<div className="flex-1 bg-neutral-900 animate-pulse" />` — full grid area
- Duration: until `next/dynamic` resolves and `mounted` guard passes
- No spinner — pulse animation only

### Stub Plugins (Phase 1 only)
- Each stub renders a placeholder card with: pane title (plugin name), dashed border, centered label "Coming in Phase 3"
- Background: `bg-surface-raised` — matches existing card pattern
- Border: `border border-dashed border-white/10`
- Label: 12px, `text-zinc-600`, centered

## Accessibility Contract

- All interactive elements have a minimum 44px touch target (icon-only buttons use `p-2` or `p-1.5` with sufficient surrounding space)
- Drag handles must have `aria-label="Drag {plugin name}"` 
- Error cards must be announced to screen readers: `role="alert"` on the error card container
- Loading skeleton must have `aria-label="Loading workspace"` and `aria-live="polite"`
- `prefers-reduced-motion`: existing `globals.css` rule disables all animations — pulse skeleton becomes static background, no behavior change needed
